# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

`goarima` is a pure-Go implementation of ARIMA (AutoRegressive Integrated Moving Average) time-series modeling. It is a library (package `goarima`) with a separate `example/` command demonstrating usage. Linear systems are solved via the external `github.com/albertyw/gaussian` package; the optional CSS and exact-MLE refinements (`refine.go`, `mle.go`) use `gonum.org/v1/gonum/optimize` for the Nelder-Mead search.

## Commands

- `make test` — full check suite: installs lint deps, runs unit tests, `go vet`, `gofmt`, `go mod tidy`, `golangci-lint`, and `govulncheck`. This is the gate that must pass before committing.
- `make unit` — unit tests only with coverage written to `c.out`.
- `make race` — tests under the race detector.
- `make cover` — runs `make test` then prints per-function coverage.
- `make benchmark` — `go test -bench=. -benchmem`.
- Run a single test: `go test -run TestHannanRissanenARMA ./...`
- Run the demo: `cd example && go run .` (runs `AutoARIMA` on several classic datasets and prints the selected orders + forecasts).
- `make example` — runs `example/compare.py`, which runs the Go demo (`go run .`), parses the orders goarima's `AutoARIMA` selected, fits `pmdarima` at those same orders, and prints the two interleaved per dataset. Needs the `example/env` Python venv; falls back to goarima-only (`go run .`) if absent. Not part of `make test`/CI.
- `make charts` — runs `example/plot_compare.py` (goarima-AutoARIMA-vs-pmdarima), `example/plot_seasonal.py` (goarima-AutoSARIMA-vs-statsmodels-SARIMAX, AirPassengers + WineInd), `example/plot_interval.py` (`ForecastInterval` prediction-interval lessons), and `example/plot_exog.py` (`WithExog`/`ForecastExog`-vs-statsmodels-SARIMAX exog) at goarima's selected orders, writing charts to the gitignored `example/charts/`. The committed copies live under `docs/images/` (refresh by copying from `example/charts/`); the seasonal, interval, and exog ones are showcased in `docs/examples.md`. Needs the `example/env` venv. Not part of `make test`/CI.

CI (`.drone.yml`) runs `make test`, `make race`, `make cover`, `make benchmark`, and `checkmake` on the Makefile.

## Architecture

Two entry points: construct a model with explicit orders via `NewARIMA(p,d,q)` + `Fit`, or let `AutoARIMA(series, maxP, maxD, maxQ)` choose the orders and return a fitted model. Full multiplicative seasonal models are available via `NewSARIMA(p,d,q,P,D,Q,m)` and `AutoSARIMA(series, maxP, maxD, maxQ, maxBigP, maxBigQ, m)`; `SeasonalOrders()` returns `(P,D,Q,m)` and `SeasonalPhi()`/`SeasonalTheta()` the seasonal AR/MA factors (validated against statsmodels SARIMAX). Both `Fit` and `AutoARIMA` are variadic in `FitOption`; `WithCSSRefinement()` enables the opt-in CSS refinement and `WithMLE()` the exact Gaussian MLE refinement (MLE takes precedence if both are passed). `WithExog(X)` adds exogenous regressors (regression with ARIMA errors, the **X** in SARIMAX — see `exog.go` and below); `Beta()` returns the fitted coefficients and `ForecastExog`/`ForecastIntervalExog` are the exog forecasters. `WithRootRepair()` reflects an unstable Hannan-Rissanen estimate back into the stationary/invertible region (root reflection, `roots.go`) instead of erroring — off by default, composes with MLE/CSS, and threads through `AutoARIMA`/`AutoSARIMA`. `Fit` and `AutoARIMA` reject a series containing NaN or ±Inf (`validateFinite`); `Forecast` errors until the model has been fitted (the `fitted` flag).

The `ARIMA.Fit` pipeline (`goarima.go`):

```
series ──Difference(d)──► y ──center (−mu)──► z (zero-mean, stationary)
   │                                            │
   └─ record anchors (last value of            └─► hannanRissanen(z,p,q)
      series differenced 0..d-1 times)              ├─ Stage 1: long AR(k) by Yule-Walker → ê
                                                     └─ Stage 2: OLS z_t ~ [z lags, ê lags] → phi, theta
                                            store lastY (p obs), lastE (q residuals), mu, anchors
```

With `WithCSSRefinement()` or `WithMLE()`, after Stage 2 the `(phi, theta)` seed is refined: `refineCSS` (`refine.go`) minimizes the conditional sum of squares, while `refineMLE` (`mle.go`) minimizes the exact Gaussian negative log-likelihood from a Kalman filter (`statespace.go`). Both run gonum Nelder-Mead through the shared `refineCoefficients` helper, penalizing non-stationary/non-invertible parameters with +Inf and falling back to the seed unless the refined fit strictly improves; residuals/`sigma2`/`lastE` are then recomputed from the refined coefficients. MLE takes precedence over CSS when both are requested.

Forecasting (`ARIMA.Forecast`) runs `forecastDiff` on the centered scale (rolling AR+MA recursion, future errors = 0), adds `mu` back, then integrates with `Undifference` once per differencing level using the stored `anchors` (a no-op when `d == 0`). `ForecastInterval` (`interval.go`) wraps `Forecast` with a two-sided prediction interval: it expands the differencing operators into the AR polynomial (`arPolynomial`), takes the MA(∞) `psiWeights`, and accumulates `Var(k) = σ²·Σψ²` (the gonum `distuv.UnitNormal` quantile sets the band), so the bounds are on the original scale and match statsmodels' `get_forecast().conf_int()`.

`AutoARIMA` (`autoarima.go` + `search.go`): `selectD` picks `d` with the KPSS level-stationarity test (`kpss.go`) — difference until the series tests stationary, up to maxD; then a search over `p,q` minimizes an information criterion (`score(crit, n, sigma2, p, q)` in `criterion.go`), skipping `(0,0)` and any fit that errors (non-stationary/non-invertible fits are rejected in `estimate.go`, see `stability.go`). The criterion defaults to AIC and is selectable with `WithCriterion(AIC|BIC|AICc)`. The search is an exhaustive grid by default; `WithStepwise()` switches to a Hyndman-Khandakar neighbor hill-climb (fewer fits, heuristic) and `WithParallel()` fits candidates concurrently (GOMAXPROCS workers, deterministic reduce — same result as serial, only faster when each fit is expensive, e.g. `WithMLE`). All three are `FitOption`s that only `AutoARIMA` reads; `Fit` ignores them.

Seasonal models (`seasonal.go`, `seasonal_arma.go`, `diff.go`): `NewSARIMA(p,d,q,P,D,Q,m)` fits the multiplicative class `φ(B)·Φₛ(Bᵐ)·(1−B)ᵈ(1−Bᵐ)ᴰ y = θ(B)·Θₛ(Bᵐ)·ε`. `Fit` applies seasonal differencing *first* (recording the last `m` values at each level in `seasonalAnchors`), then the regular-diff/center pipeline runs on `s`; the ARMA estimator is `seasonalHannanRissanen` (HR extended with seasonal-lag regressors as the seed). The seasonal AR/MA factors `Φₛ`/`Θₛ` are stored separately from the regular `φ`/`θ`, and `expandSeasonalAR`/`expandSeasonalMA` (`seasonal_arma.go`, via `polyMul`) multiply each pair into the *expanded* AR/MA polynomials (`expandedPhi`/`expandedTheta`) that `forecastDiff`, the state-space build, and `ForecastInterval` all run on unchanged. `WithMLE`/`WithCSSRefinement` refine over the packed factor vector `(φ,Φₛ,θ,Θₛ)` via `refineSeasonalMLE`/`refineSeasonalCSS` (expanding inside the objective; per-factor stability penalty); the non-seasonal refiners are the `P=Q=0` special case. `Forecast` undoes regular then seasonal differencing (`SeasonalUndifference`). `AutoSARIMA` picks `D` (0 or 1) with the Wang-Smith-Hyndman seasonal-strength measure `Fs` (`selectSeasonalD`/`seasonalStrength`/`centeredMovingAverage` in `seasonal.go`, threshold 0.64, matching R `forecast::nsdiffs(test="seas")`), then `d` via KPSS on `s`, then `p,q,P,Q` via the 4-D `searchSpace` (which carries `bigD`/`period`/`maxBigP`/`maxBigQ`). All `AutoARIMA` `FitOption`s thread through `AutoSARIMA` identically.

Exogenous regressors / SARIMAX (`exog.go`): `WithExog(X)` supplies an `n×k` regressor matrix and fits *regression with ARIMA errors* `y_t = X_t·β + η_t` (not ARIMAX — no lagged X), matching statsmodels SARIMAX `exog=` with `trend="n"`. `Fit` estimates β by a two-step OLS (`estimateExogBeta`: difference `y` and each column of `X` by the model's seasonal-then-regular orders via `differenceExog`, solve the normal equations with `olsBeta`/`gaussian.Solve`), then runs the *entire* existing pipeline on the level-scale residual series `η = y − Xβ` (`fitSeries`). Under `WithMLE`/`WithCSSRefinement`, β is refined *jointly* with the ARMA factors: `refineExog` packs `(β,φ,Φₛ,θ,Θₛ)` into the Nelder-Mead vector, re-forming `η` and re-centering (`centeredDiff`) each trial, then rebuilds `fitSeries`/anchors/`z`/residuals from the refined β. Forecasts add the future regression mean back: `Forecast`/`ForecastInterval` guard `exogDim>0` and error (use `ForecastExog`/`ForecastIntervalExog`), which call the shared `forecastLevel`/`forecastIntervalCore` cores and shift by `exogMean(futureX,β)` (`futureX` is `h×k`); intervals exclude β uncertainty (matches statsmodels' default `conf_int`). `AutoARIMA`/`AutoSARIMA` validate the matrix, net X out with a provisional β (`olsBeta` on the level series) before `selectD`/`selectSeasonalD`, then re-estimate β per candidate fit (`WithExog` threads through `opts`). The `beta`/`exogDim` struct fields are nil/0 without exog, so the no-exog path is unchanged.

Key files:
- `exog.go` — `WithExog`, `validateExogMatrix`, `differenceExog`/`exogColumn`, `olsBeta`, `regressionResiduals`, `estimateExogBeta`, `centeredDiff`, `refineExog` (joint β+ARMA refinement), `exogMean`, `ForecastExog`, `ForecastIntervalExog`.
- `goarima.go` — `ARIMA` struct (incl. `bigP`/`bigD`/`bigQ`/`period`, `seasonalPhi`/`seasonalTheta`, cached `expandedPhi`/`expandedTheta`, `seasonalAnchors`, `beta`/`exogDim`), `NewARIMA`, `NewSARIMA`, `Fit` (with the exog branch), `Forecast`/`forecastLevel`/`forecastDiff`, getters (`Orders`/`SeasonalOrders`/`SeasonalPhi`/`SeasonalTheta`/`Beta`/…), `mean`/`meanSquare`/`isConstant`.
- `seasonal_arma.go` — `expandSeasonalAR`/`expandSeasonalMA` (multiplicative factor expansion via `polyMul`) and `seasonalHannanRissanen` (the seasonal seed estimator).
- `interval.go` — `ForecastInterval` (exog guard) + `forecastIntervalCore` and the `Forecast` result struct; `psiWeights`/`arPolynomial`/`polyMul` for the MA(∞) forecast-error variance.
- `diff.go` — `Difference` / `Undifference`, `SeasonalDifference` / `SeasonalUndifference`.
- `estimate.go` — `hannanRissanen` (the ARMA estimator), `hrAROrder`, `armaResiduals`.
- `refine.go` — `nelderMeadRefine` (the shared gonum Nelder-Mead core), `refineCoefficients`/`refineSeasonalCoefficients` (pack/unpack the factor vector), and `refineCSS`/`refineSeasonalCSS` (conditional-sum-of-squares refinement).
- `mle.go` — `refineMLE`/`refineSeasonalMLE`, the opt-in exact Gaussian maximum-likelihood refinement (Kalman-filter likelihood over the expanded polynomials, gonum Nelder-Mead).
- `statespace.go` — `buildStateSpace` (Harvey ARMA state-space form), `solveLyapunov` (stationary init via the discrete Lyapunov equation), `kalmanConcentratedNLL` (Kalman prediction-error decomposition, concentrated negative log-likelihood).
- `stability.go` — `isStationary` / `isInvertible` (reflection-coefficient root test), used by the fit guards and the refinement penalty.
- `roots.go` — `polyRoots` (companion-matrix eigenvalues via gonum/mat), `reflectUnstableRoots`, and `repairStationary`/`repairInvertible` (root-reflection repair for `WithRootRepair`).
- `kpss.go` — `kpssLevelStationary`, the KPSS level-stationarity test used by `selectD`.
- `yulewalker.go` — autocovariance helpers and `solveYuleWalker` / `solveYuleWalkerFromAutocov` (used as the Hannan-Rissanen Stage 1 AR fit; guards constant series).
- `autoarima.go` — `AutoARIMA`, `AutoSARIMA`, `selectD` (order-search dispatch + final refit).
- `seasonal.go` — `selectSeasonalD` (seasonal-strength `Fs` test for D), `seasonalStrength`, `centeredMovingAverage`, `variance`.
- `criterion.go` — `Criterion` (AIC/BIC/AICc) and `score`, the information criterion AutoARIMA minimizes.
- `search.go` — `searchSpace` (carries `bigD`/`period`/`maxBigP`/`maxBigQ`), the `order` `[4]int` `(p,q,P,Q)`, `evalCandidate`/`evalBatch` (optionally parallel), `gridSearch`, and `stepwiseSearch` (Hyndman-Khandakar) — all over the 4-D `(p,q,P,Q)` space.

By default, estimation is Hannan-Rissanen (pure linear algebra, no optimizer); `WithCSSRefinement()` adds a least-squares (CSS) refinement and `WithMLE()` an exact Gaussian maximum-likelihood refinement (Kalman filter, matching statsmodels' `method="statespace"`). The HR default and CSS refinement are approximate; MLE is the exact-likelihood fit, though small numeric differences from statsmodels remain.

The `ARIMA` struct fields are unexported; access state through the getter methods (`Orders`, `Phi`, `Theta`, `Beta`, `LastY`, `LastE`, `LastOrig`, `Sigma2`). The slice getters (`Phi`, `Theta`, `Beta`, `LastY`, `LastE`) return copies (`copyFloats`), so callers cannot mutate internal model state.

## Example data & reference scripts

- `example/main.go` runs `AutoARIMA` on several `example/data/*.csv` datasets (newline-separated values): airpassengers, lynx, wineind, sunspots, woolyrnq, austres. `runAuto` selects the order via `AutoARIMA` (fast HR path) then refits that order with `WithMLE()`, so the reported coefficients are maximum-likelihood — without that the Hannan-Rissanen seed shown would diverge from pmdarima's MLE for hard, weakly-identified orders (e.g. wineind's near-unit-root (3,1,3)). It prints one `[goarima] <name> ARIMA(p,d,q)` block per dataset (order + phi/theta/forecast); `compare.py` and `plot_compare.py` parse those blocks. It also prints `[goarima-seasonal] <name> ARIMA(p,d,q)(P,D,Q)[m]` blocks from `AutoSARIMA` (with `maxBigP=maxBigQ=1`) for AirPassengers and WineInd (distinct prefix so `compare.py`, which matches `[goarima]`, ignores them; `plot_seasonal.py` parses them). It also prints `[goarima-interval] <key> <order>` blocks (a point forecast plus `lower@L`/`upper@L` bands) from `ForecastInterval` — non-seasonal vs seasonal AirPassengers at 95%, and Lynx at 80%+95% — which `plot_interval.py` parses. It also prints a `[goarima-exog] <name> <order>` block (estimated `beta` + a `ForecastExog`) from a deterministic covariate-driven synthetic series (`exogExample`/`printExog`), which `plot_exog.py` parses (regenerating the same series from the identical closed form). CSVs are exported from the vendored pmdarima `*.py` generators (or statsmodels for sunspots).
- `integration_test.go` is the rigorous integration suite. It lives in the **external `goarima_test` package** so it exercises only the exported API. It embeds `example/data/*.csv` and the committed JSON fixtures and runs four tiers (no network/Python at test time): Tier 1a fixed-order fits vs pmdarima (`TestFixedOrdersMatchPmdarima` — coefficients only for pure AR/MA, forecasts only for `d==0`, because mixed-ARMA coefficients are non-identifiable and goarima adds a drift for `d>=1`); Tier 1b `auto_arima` selection vs pmdarima (`TestAutoSelectionVsPmdarima` — same `d`, p/q not required to match); Tier 2 analytic closed-forms via the public API; Tier 3 the goarima golden baseline (`TestGoldenWithMLE`). Seasonal models add three more: `TestSeasonalFixedOrderMatchesPmdarima` (goarima `NewSARIMA` vs a statsmodels **SARIMAX** `seasonal_order=(0,D,0,m)` fit — AirPassengers `(1,1,0)(0,1,0)12`; phi only, since `d>=1` forecasts carry the drift gap), `TestSeasonalARMAMatchesStatsmodels` (the airline model `(0,1,1)(0,1,1)12` vs SARIMAX — regular + seasonal MA factors compared under `WithMLE`), and `TestGoldenAutoSeasonalSelection` (an `AutoSARIMA` golden lock with `maxBigP=maxBigQ=1` under the `auto_seasonal` key, which selects a seasonal-AR term). `TestForecastIntervalMatchesStatsmodels` checks `ForecastInterval` against statsmodels' `get_forecast().conf_int()` (AirPassengers `(2,1,0)`; interval *half-widths* compared, isolating the σ²·Σψ² variance from the `d>=1` drift). `TestExogMatchesStatsmodels` checks regression with ARIMA errors against statsmodels SARIMAX `exog=` (a synthetic `(1,0,0)` series embedded in the `exog` fixture: β and forecast levels compared directly since `d==0`, plus `ForecastIntervalExog` half-widths).
- `testdata/pmdarima_reference.json` — committed pmdarima fixtures (fixed-order + `auto_arima` + `seasonal_fixed` (SARIMAX `seasonal_order=(0,D,0,m)`) + `seasonal_arma` (SARIMAX with nonzero P/Q — the airline model, four factors captured separately) + `interval` (statsmodels forecast confidence bounds) + `exog` (SARIMAX `exog=` regression with ARIMA errors — embedded `x`/`y`/`future_x` plus β/φ/θ/forecast/conf_int)), regenerated by `example/gen_reference.py` (`cd example && env/bin/python gen_reference.py`).
- `testdata/goarima_golden.json` — committed goarima `WithMLE` baseline (plus the `auto_seasonal` `AutoSARIMA` lock), regenerated by `go test -run TestGoldenWithMLE -update` (normal runs never write it).
- `example/compare.py` is the human eyeball comparison driver (run via `make example`): it runs the Go demo, parses the orders goarima's `AutoARIMA` chose, fits `pmdarima` at those same orders (pmdarima's default intercept adds a drift for `d>=1`, matching goarima), and prints the two interleaved per dataset. It depends on the `report` output format in `main.go` (the `[goarima] … ARIMA(p,d,q)` blocks), so keep that format and `parse_blocks` in sync.
- `example/plot_compare.py` (`make charts`) reuses `compare.py`'s helpers to draw the same goarima-AutoARIMA-vs-pmdarima comparison as charts.
- `example/plot_seasonal.py` (`make charts`) parses the `[goarima-seasonal]` blocks, fits a statsmodels **SARIMAX** at the same `order`/`seasonal_order`, and renders the seasonal forecast charts (`docs/images/{airpassengers,wineind}_seasonal.png`) shown in `docs/examples.md`.
- `example/plot_interval.py` (`make charts`) parses the `[goarima-interval]` blocks and renders two prediction-interval lesson charts (`docs/images/{airpassengers,lynx}_interval.png`): AirPassengers non-seasonal-vs-seasonal 95% bands (with pmdarima overlay, showing a smaller σ² tightens the band) and Lynx nested 80%/95% bands.
- `example/plot_exog.py` (`make charts`) parses the `[goarima-exog]` block, regenerates the same covariate-driven series from the closed form (kept in sync with `main.go`'s `exogExample`), fits a statsmodels SARIMAX `exog=` at the same order, and renders `docs/images/exog.png` (goarima exog vs SARIMAX exog vs a no-exog ARIMA that reverts to the mean).
- Order sync: the **demo** (`main.go`/`compare.py`/`plot_compare.py`) uses goarima's *auto-selected* orders, so nothing is hard-coded there. The **integration test** still uses *fixed* orders — those live in `example/gen_reference.py` and must stay in sync with the committed `testdata/*.json` fixtures (regenerate both when they change).
- Python deps are pinned in `example/pyproject.toml` and installed under `example/env`.
